Insert

The block

The insert block is the block you want to choose, if you want to insert values in your database. It symbolises the insert structure. Blocks, like tables and operators are plugged in it.
It also can be mutated.
If you want to learn more about how to mutate the block please, see 'Functionality'.
The not mutated select has one input in the 'INTO'-section. This block also changes the colour, to symbolise the types used in SQL.

Functionality

To mutate the block, you click on the star-icon and a new workspace appears.

In this workspace you can drop blocks into an 'ADD'-block to expand the insert-block. So for each droped block one new input appears. Like you see in the picture above.
If you want to reduce the number of inputs, you click on the star icon and pull the blocks out of the 'ADD' block.
The block in this workspace symbolises the'INTO' statement. In the section 'Statement' you will learn more about them.

Statement

In the mutator workspace there ist one block symbolising the 'INTO'-statement in SQL.

INTO

The 'INTO' input uses the compare operator. Into the first input of the compare operator, you need to plug in the tables and columns value, used here as the insert destination. As second input value your need a value, corresponding in type and colour to the first input.
This is shown in the picture above. Be sure to use only tables and columns you use in the 'INSERT'-section,
otherwise there will be a warning.
Be sure the you use only the same table in the insert Block. The usage of different Tables is not provided.

The types

This colour gradient, apple green to cyan, symbolises a list of columns of the database

This colour gradient, violet to cyan, symbolises the numeric values of the database

This colour gradient, brown to cyan, symbolises the textual values of the database

This colour gradient ,tapestry to cyan, symbolises the date values of the database

This colour ,cyan, symbolises the boolean values of the database
