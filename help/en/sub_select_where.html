Subselect (section 'WHERE')

The block

The subselect block is used in the 'WHERE'-section of the 'SELECT' Block with the compare operator. It symbolises the subquery structure. All the other blocks, like tables, functions and operators are plugged in it.
It also can be mutated.
If you want to learn more about how to mutate the block please, see 'Functionality'.
The not mutated subselect block is divided in two sections. The 'SELECT'-section and the 'WHERE'-section. If you want to leran more about the sections please take a look at the 'SELECT' block. This block also changes the colour, to symbolize the types used in SQL.

Functionality

To mutate the block, you click on the star-icon and a new workspace appears.

In this workspace you can drop blocks into an 'ADD'-block to expand the select-block. So for each droped block one new input appears. Like you see in the picture above.
If you want to reduce the number of inputs, you click on the star icon and pull the blocks out of the 'ADD' block.
The blocks in this workspace are the SQL clauses 'GROUP BY', GROUP BY' with 'HAVING, 'ORDER BY', 'LIMIT' and an 'ALIAS' . In the section 'Clauses' you will learn more about them.

Clauses

In the mutator workspace there are five blocks, four of them symbolising the clauses, 'GROUP BY',GROUP BY' with 'HAVING, 'ORDER BY' and 'LIMIT', in SQL and one symbolises the 'ALIAS'. Notice that the first three clauses could be used with 'LIMIT' and 'ALIAS' but not with each other. 'LIMIT' and 'ALIAS' can also be used alone.

Alias

The 'ALIAS' textfield is used if you want to give this subselect a special name in the result. To use the name later there is a alias variable.

GROUP BY

The 'GROUP BY' input is used if you use a groupfunction. If you use 'GROUP BY' be sure to use the same tables in the group, which you used in the select.
Notice that only the table should be used, which are not in a groupfunction.
You can use this clause with the 'LIMIT' clause

GROUP BY with HAVING

The 'GROUP BY' with 'HAVING' input is used if you use a groupfunction and want to requiere the order. If you use 'GROUP BY' be sure to use the same tables in the group, which you used in the select.
Notice that only the table should be used which are not in a groupfunction.
In the 'HAVING' input you need to use a compare operator.
You can use this clause with the 'LIMIT' clause

ORDER BY

The 'ORDER BY' input is used if you want to order your result after a specific order. If you use 'ORDER BY' be sure to use one of tables in the 'ORDER BY', which is also used in the select. You also can use the alias variable here. You can sort your results ascendant or descandent.
You can use this clause with the 'LIMIT' clause.

LIMIT

The 'Limit' input is used if you want to limit your result after. It is used with a number variable and you can use tit with the other clauses
