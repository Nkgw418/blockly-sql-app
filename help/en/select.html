Select

The block

The select block is the block you need to choose, if you want to do a query on you database. It symbolises the query structure. All the other blocks, like tables, functions and operators are plugged in it.
It also can be mutated.
If you want to learn more about how to mutate the block please, see 'Functionality'.
The not mutated select block is divided in two sections. The 'SELECT'-section and the 'WHERE'-section. This block also changes the colour, to symbolise the types used in SQL.

Functionality

'SELECT'-section

Into the 'SELECT'-section you plug all the table and column blocks , which you want to query. You can also blug in the 'SUBSELECT' Block and the groupfunction.
A simple example is shown above.

'WHERE'-section

Into the 'WHERE'-section you plug the blocks which require the query. Into the 'WHERE' input you can plug the 'AND'-block and the compare operator. Every other block to require the search will be plugged into one of the blocks.
An example is shown above.

Mutation

To mutate the block, you click on the star-icon and a new workspace appears.

In this workspace you can drop blocks into an 'ADD'-block to expand the select-block. So for each droped block one new input appears. Like you see in the picture above.
If you want to reduce the number of inputs, you click on the star icon and pull the blocks out of the 'ADD' block.
The blocks in this workspace are the SQL clauses 'GROUP BY', GROUP BY' with 'HAVING, 'ORDER BY' and 'LIMIT'. In the section 'Clauses' you will learn more about them.

Clauses

In the mutator workspace are four blocks, symbolizing the clauses, 'GROUP BY',GROUP BY' with 'HAVING, 'ORDER BY' and 'LIMIT', in SQL. Notice that the first three clauses could be used with 'LIMIT' but not with each other. 'LIMIT' could be also used alone.

GROUP BY

The 'GROUP BY' input is used if you use a groupfunction. If you use 'GROUP BY' be sure to use the same tables in the group, which you used in the select.
Notice that only the tables should be used, which are not in a groupfunction.
You can use this clause with the 'LIMIT' clause

GROUP BY with HAVING

The 'GROUP BY' with 'HAVING' input is used if you use a groupfunction and want to requiere the order. If you use 'GROUP BY' be sure to use the same tables in the group, which you used in the select.
Notice that only the tables should be used which are not in a groupfunction.
In the 'HAVING' input you need to use a compare operator.
You can use this clause with the 'LIMIT' clause

ORDER BY

The 'ORDER BY' input is used if you want to order your result after a specific order. If you use 'ORDER BY' be sure to use one of the tables in the 'ORDER BY', which is also used in the select. You also can use the alias variable here. You can sort your results ascendant or descandent.
You can use this clause with the 'LIMIT' clause

LIMIT

The 'Limit' input is used if you want to limit your result after. It is used with a number variable and you can use it with the other clauses.
