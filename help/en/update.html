Update

The block

The update-block is the block you want to choose, if you want to update specific values in your database. It symbolises the update structure.
Blocks, like tables and operators are plugged in it. It also can be mutated.
If you want to learn more about how to mutate the block please, see 'Functionality'.
The not mutated update-block is divided in three sections. The 'UPDATE'-section, the 'SET'-section and the 'WHERE'-section.

Functionality

To mutate the block, you click on the star-icon and a new workspace appears.

In this workspace you can drop blocks into an 'ADD'-block to expand the update-block. So for each droped block a new input appears. Like you see in the picture above.
If you want to reduce the number of inputs, you click on the star icon and pull the blocks out of the 'ADD' block.
The block in this workspace symbolises the'SET' statement. In the section 'Statement' you will learn more about it.
In the 'WHERE'-section you require your update statement. This section is used like it is used in the 'SELECT' Block.

Be sure the you use only the same table in the insert Block. The usage of different Tables is not provided.

Statement

In the mutator workspace there is one block symbolising the 'SET'-statement in SQL.

SET

The 'SET' input uses the to-operator, as first input, in the compare, the tables and columns value as the update destination and as second input a value, corresponding in type and colour to the first input.
