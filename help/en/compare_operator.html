Compare operator

The block

This block symbolises the compare operations in SQL. The compare operation compares two expressions of the same type, for example a numeric value with a numeric value or a number function with a numeric value.
The compare operator returns true, if the compare is successful, and false, if not.
It is used either in the "WHERE"-section directly or with the 'AND'-block or 'OR'-block . You can also use it in the "HAVING"-statement of the "GROUP BY" clause.
There are four colour gradiens, symbolizing the type, which you are comparing.

Functionality

In this block is one dropdown holding the various compare operations. So you can choose your compare operation from the dropdown as you see on the left.
Depending on which type of value you are plugging in the first input, the compare operator will change it's colour to a colour gradient,as you see in th picture below.
For more information on the meaning of the colour gradient see 'The types'.

Please make sure that the second value, which you want to compare, has the same type ( the same colour) as the first one. If not the block will pop out.

One special compare operation, in terms of colouring and corresponding values, is 'like'. If you choose like in the dropdown, the block will change the colour and, if there are blocks, unplug all blocks which are not from the textual type(colour brown).

The types

This colour gradient , cyan to violet, symbolises a numeric value compare.

This colour gradient, cyan to brown, symbolises a textual value compare.

This colour gradient,cyan to tapestry, symbolises a date value compare.

This colour ,cyan, symbolises a boolean value compare
